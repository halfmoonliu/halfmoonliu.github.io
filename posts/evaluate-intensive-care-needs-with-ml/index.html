Evaluate Intensive Care Needs with ML

Machine learning (ML) can be applied in hospital to improve health care quality. This is a demonstration of steps to build ML models to evaluate the need for intensive care within the first 24 hours of admission. The post includes the workflow from data cleansing, feature engineering and model building that can be direcly applied in real-world settings.

Quick Summary üîó

Computer-aided evaluation of patients severity can improve health care quality in hospitals. This project demonstates the workflow of building ML models to evaluate the need for intensive care within first day of admission using randomly generated Electronic Health Records(EHRs). A quick project walkthrough slide can be found here. Below are details of key steps of the workflow.

Step 1: Create Dataset üîó

To demostrate the workflow, randomly generated EHRs were used. Items related to creating a disease specific severity evaluation model, such as sex, age, diagnosis code, vital signs, lab exams…etc were included. The generated data imitates the format of real-world EHRs extracted from health information systems.

Step 2: Select Cohort with Specific Disease üîó

Since the condition of patients with different diseases can be reflected on different physical indices, the first step of building a alarm system is to select a cohort for model building. Here, we select the cohort by thier ICD (International Statistical Classification of Diseases and Related Health Problems) code.
ICD code are detail classification of existing diseases that evloves through time. Thus, multiple ICD codes can be grouped as one disease. For example, the ICD code 485 (brochopenumonia, organism unspecified) and 486 (pneumonia, organism unspecified) can both be considered pneumonia related codes. As time goes, new versions of ICD will replace the older version.485, 486 are codes from ICD-9; the corresponding codes in ICD-10 are J18.0(Bronchopneumonia, unspecified organism), J18.8(Other pneumonia, unspecified organism), and J18.9(Pneumonia, unspecified organism). As one can notice by glancing the codes, similar diseases have similar ICD codes. In ICD-10, codes begining with J1 are all pneumonia or respiratory tract infection related diseases; codes that begin with J18 are different kinds of pneumonia, unspecified organism.

Step 3: Data Preprocessing and Feature Engineering üîó

a. Cormorbid condition:
Cormorbid conditions can have huge impact on prornosis. For example, pneumonia patients with cardiovascular diseases are of higher risk of ICU admission. Thus, we can group diagnosis code to a numbers of cormorbid conditions (i.e. if a patient has any of the diagsnosis code A, B or C, he/she is considered having Cormorbid Condition 1). In the demo, five cormorbid condtions were put into the model as features (with/without the cormorbid condtion).
b. Pathogen:
For infectious disease, pathogens are related to the evaluation of severity. For example, bacterial pneumonia tends to be more severe in children than viral pneumonia. In this demo, regular expression was used to extract positive pathogen exam findings and add the finding to ML model as one feature (with/ without positive findings of the pathogen).
c. Lab data:
Various biochemical test conducted in labs can reveal patient’s general body function (e.g. white blood cell count, C-Reactive Protein…etc). To evaluate the need for intensive care of inpatient on the first day, initial records of lab data were included as features for machine learning models.
d. Vital signs:
Vital signs such as temperature, pulse or blood pressure are important indicators patient condition. In this demo, maximum temperature, pulse, minimum systolic and diastolic pressure recorded within the first day of admission was included as features in the model to predict ICU admission.

Step 4: Model Selection and Hyperparameter Tuning: üîó

In an outcome prediction task where few features are of fine temporal resolution, tres-based methods, such as XGboost and random forest, can yield good performances. In these models, maximum depth of trees, learning rate and regularization are some of the hyperparameters that can further enehance model prediction power. ML models can outperform existing indicies or severity score that clinician used nowadays, which come from previous studies using logistic regression models. In the demo, six machine learning models were chosen for the prediction task: Boosted Trees, Random Forest, Feedforward Neural Network, Logistic Regreesion, Support Vector Machines (SVM) and K-Nearest Neighbors (KNN).
